# /klw:query — 查询知识库

对 LLM Wiki 知识库提出问题，综合多个页面信息回答。

## 知识库路径

知识库路径：`{knowledgeBase}`（记为 `$KB`）

## 流程

### Step 1: 理解问题

明确用户的问题是什么，涉及哪些主题、实体或概念。

### Step 2: 搜索索引

读取 `$KB/wiki/index.md`，找到与问题相关的页面。根据问题类型选择搜索策略：
- **实体相关**：查 `$KB/wiki/entities/`
- **概念相关**：查 `$KB/wiki/concepts/`
- **来源相关**：查 `$KB/wiki/sources/`
- **跨类别**：查所有相关页面

### Step 3: 深入阅读

读取找到的相关页面，提取与问题相关的信息。注意交叉引用，可能需要追踪 `[[链接]]` 到其他页面。

### Step 4: 综合回答

基于 wiki 内容生成答案：
- 直接回答问题
- 附带 `[[页面名]]` 引用，说明信息来源
- 如果多个页面有矛盾信息，指出差异
- 如果 wiki 中信息不足，明确说明

### Step 5: 可选 — 归档答案

如果查询答案本身具有长期价值（如比较分析、综合研究、关联发现），询问用户是否将其归档为新的 wiki 页面：
- 归档到 `$KB/wiki/sources/`（如果是综合分析）
- 归档到 `$KB/wiki/concepts/`（如果是概念性总结）
- 归档到 `$KB/wiki/entities/`（如果是实体间的关联分析）

归档后更新 `$KB/wiki/index.md` 和 `$KB/wiki/log.md`。

### Step 6: 记录日志

在 `$KB/wiki/log.md` 末尾追加：

```markdown
## [YYYY-MM-DD] query | 问题简述

- 查询涉及页面：[[页面1]]、[[页面2]]
- 归档答案：是/否
```
