# /klw:lint — 知识库健康检查

对 LLM Wiki 进行全面健康检查，发现问题并给出修复建议。

## 知识库路径

知识库路径：`{knowledgeBase}`（记为 `$KB`）

## 流程

### Step 1: 加载索引

读取 `$KB/wiki/index.md` 获取所有页面列表。

### Step 2: 逐项检查

**矛盾检测**
- 读取所有页面，检查是否存在相互矛盾的信息
- 特别关注实体页面和概念页面之间的不一致
- 记录矛盾位置和具体内容

**过时检测**
- 检查页面的 `date` 字段，标记较早的页面
- 对比较新的来源摘要，看是否有信息已被取代
- 标记可能需要更新的页面

**孤立检测**
- 检查哪些页面没有被其他页面通过 `[[链接]]` 引用
- 这些页面可能是冗余的，或者缺少从其他页面的关联

**缺口检测**
- 检查是否有页面中提到了某个概念/实体但没有独立页面
- 检查是否有重要主题缺少覆盖
- 建议可以创建的新页面

**链接完整性**
- 检查 `[[链接]]` 指向的页面是否实际存在
- 标记断开的链接

**索引一致性**
- 检查 `$KB/wiki/index.md` 是否包含了所有实际存在的页面
- 检查是否有页面存在但未被索引

### Step 3: 呈现报告

```
知识库健康检查报告
==================

页面总数：N
检查时间：YYYY-MM-DD

问题汇总：
  矛盾：X 处
  过时：X 个页面
  孤立：X 个页面
  缺口：X 个概念
  断链：X 处
  索引遗漏：X 个页面

详细问题：
...

修复建议：
1. [高优先级] ...
2. [中优先级] ...
3. [低优先级] ...

建议下一步：
- 可以摄入的新来源：...
- 可以创建的新页面：...
- 需要更新的页面：...
```

### Step 4: 记录日志

在 `$KB/wiki/log.md` 末尾追加：

```markdown
## [YYYY-MM-DD] lint | 健康检查

- 页面总数：N
- 发现问题：矛盾 X 处、过时 X 个、孤立 X 个、缺口 X 个、断链 X 处
- 建议操作：...
```

## 重要规则

1. **只报告，不自动修复** —— 列出问题和建议，由用户决定是否修复
2. **按优先级排序** —— 矛盾和断链优先，孤立和缺口次之
3. **给出具体位置** —— 每个问题都要说明在哪个页面的哪个部分
